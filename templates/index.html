Deep Knowledge Base
Does UrlForge work on JavaScript-heavy SPAs built with React or Next.js?
Yes. The crawler uses Playwright with a headless Chromium browser, so it fully renders JavaScript before extracting links, meta tags, and content — the same way Google's crawler does. Static HTML sites are also supported and crawl faster.
Do I need an OpenAI API key to use UrlForge?
No. UrlForge supports four LLM providers (OpenAI, Gemini, OpenRouter, and Ollama for fully local inference) and automatically falls back through the chain if one isn't configured. If no API key is set at all, the content engine uses a built-in heuristic fallback with TF-IDF and PMI phrase analysis — so you get useful output at zero API cost.
How long does a full site audit take?
Small sites (under 50 pages) typically complete in under 2 minutes. Larger sites depend on concurrency settings and server response times. The crawler uses an async worker pool, so it processes multiple pages in parallel. You can watch real-time progress stream to the UI while it runs.
Will UrlForge crawl pages that require a login?
Yes — you can provide HTTP Basic Auth credentials or a Bearer Token in the crawl configuration, and the crawler will include those headers on every request. Full session-based auth (form login flows) is not currently supported.
Can I use UrlForge without deploying it — just to generate a sitemap or audit report?
Absolutely. Deployment is optional. You can use UrlForge purely as an audit tool — crawl a site, review the 20-module report, download the PDF and generated sitemap.xml, and do nothing else. The deployment targets (GitHub, Vercel, FTP, etc.) are only triggered when you explicitly approve and push changes.
How does the self-healing CI/CD work in practice?
After deploying to GitHub, UrlForge polls your GitHub Actions workflow until it completes. If it fails, the engine fetches the raw failure logs, sends them along with the deployed code to your configured LLM, gets a corrected version back, commits it, and triggers a new deployment — looping until the workflow passes or the retry limit is hit.
Is the generated content original, or will it get flagged as AI-written?
UrlForge grounds all content generation in your site's actual data — services, brand voice, technologies, and audience — extracted by the business intelligence crawler. It also applies anti-AI-trope filters that actively reject generic filler phrases. The result is content that reflects your actual brand rather than generic LLM output. That said, we always recommend human review before publishing.
Does it respect robots.txt? Will it overload my server?
Yes, robots.txt compliance is enforced. The crawler also limits concurrency and enforces configurable request timeouts, so it won't hammer your server. Only same-domain links are followed by default, preventing accidental crawls of third-party resources.
Can I run UrlForge locally for privacy?
Yes. UrlForge runs fully locally with Docker or a plain Python venv. Pair it with Ollama for local LLM inference and no data leaves your machine at all — no crawl data, no content, no site URLs are sent to any external service.
What's the difference between a standard crawl and the plugin workflow?
A standard crawl gives you an immediate audit report — scores, issues, sitemap, and a suggested action list. The plugin workflow adds an approval gate: it runs the full pipeline (crawl → analyze → generate fixes → generate new content), shows you a before/after SEO score comparison, lets you review and edit individual fixes or FAQ entries, and only deploys when you explicitly approve. It's designed for agency client reviews or team collaboration.
